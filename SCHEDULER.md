# 定时任务功能

微信 Kimi Bot 支持通过 crontab 表达式定时触发执行任务。

## 功能特性

- ✅ 支持标准 crontab 表达式（5个字段：分 时 日 月 周）
- ✅ 任务持久化存储
- ✅ 支持通过聊天命令管理任务
- ✅ 支持通过命令行工具管理任务
- ✅ 任务执行结果自动发送到微信聊天

## 快速开始

### 1. 自然语言创建任务（推荐）

用日常语言描述你想创建的定时任务：

```
/task create 每天早上9点搜集AI资讯
```

机器人会解析你的描述，然后请你确认：

```
🤖 解析结果

任务名称: 每日AI资讯
执行时间: 每天早上 9:00
Crontab: 0 9 * * *
执行命令: kimi --quiet --prompt '搜集今日AI新闻并总结'

回复 "确认" 创建此任务，或 "取消" 放弃
(5分钟内有效)
```

回复 **"确认"** 即可创建任务，或 **"取消"** 放弃。

### 2. 添加示例任务

```
/task example
```

这将添加一个每天早上9点搜集AI资讯的定时任务。

### 2. 查看任务列表

```
/task list
```

### 3. 立即执行任务（测试）

```
/task run <任务ID>
```

## Crontab 表达式格式

格式：`分 时 日 月 周`

| 字段 | 范围 | 说明 |
|------|------|------|
| 分   | 0-59 | 分钟 |
| 时   | 0-23 | 小时 |
| 日   | 1-31 | 日期 |
| 月   | 1-12 | 月份 |
| 周   | 0-6  | 星期（0=周日）|

### 常用示例

| 表达式 | 说明 |
|--------|------|
| `0 9 * * *` | 每天早上 9:00 |
| `0 8 * * *` | 每天早上 8:00 |
| `0 0 * * *` | 每天凌晨 0:00 |
| `0 */6 * * *` | 每 6 小时 |
| `0 */12 * * *` | 每 12 小时 |
| `0 9 * * 1` | 每周一早上 9:00 |
| `0 9 * * 1-5` | 工作日早上 9:00 |
| `0 0 1 * *` | 每月 1 号凌晨 0:00 |
| `0 0 * * 0` | 每周日凌晨 0:00 |

### 特殊符号

- `*` - 任意值
- `-` - 范围，如 `1-5` 表示1到5
- `,` - 列表，如 `1,3,5` 表示1、3、5
- `/` - 步长，如 `*/5` 表示每5个单位

## 聊天命令

### 自然语言创建任务

```
/task create <自然语言描述>
```

示例：
```
/task create 每天早上9点搜集AI资讯
/task create 每工作日早上8点半提醒我打卡
/task create 每周日晚上8点总结下周计划
/task create 每月1号早上发月度总结
```

机器人会解析你的描述，然后请你确认后再创建任务。

### 精确添加任务

如果你熟悉 crontab，可以直接指定：

```
/task add "任务名称" "crontab" "执行命令"
```

示例：
```
/task add "每日早报" "0 8 * * *" "kimi --quiet --prompt '总结今日新闻'"
```

### 删除任务

```
/task del <任务ID>
```

### 启用/禁用任务

```
/task toggle <任务ID>
```

### 立即执行任务

```
/task run <任务ID>
```

### 列出所有任务

```
/task list
```

### 添加示例任务

```
/task example
```

## 命令行工具

除了聊天命令，也可以使用命令行管理定时任务：

```bash
# 显示所有任务
npm run schedule

# 交互式添加任务
npm run schedule -- --add

# 删除任务
npm run schedule -- --delete <任务ID>

# 启用/禁用任务
npm run schedule -- --toggle <任务ID>

# 添加示例任务
npm run schedule -- --example

# 显示帮助
npm run schedule -- --help
```

## 任务存储

每个Agent的定时任务独立存储：
- **多Agent模式**: `~/.weixin-kimi-bot/agents/{agent_id}/scheduled-tasks.json`
- 每个Agent拥有完全独立的定时任务
- 可以手动编辑（需要重启Agent生效）

## 使用场景示例

### 1. 每日AI资讯

每天早上9点自动搜集AI资讯：

```
/task add "每日AI资讯" "0 9 * * *" "kimi --quiet --prompt '搜集今日AI新闻'"
```

### 2. 定时提醒

每天中午12点提醒吃饭：

```
/task add "午餐提醒" "0 12 * * *" "echo '⏰ 该吃午饭啦！'"
```

### 3. 工作日早报

工作日早上8:30发送早报：

```
/task add "工作日早报" "30 8 * * 1-5" "kimi --quiet --prompt '总结今日科技新闻'"
```

### 4. 每周总结

每周五下午6点生成本周工作总结：

```
/task add "周总结" "0 18 * * 5" "kimi --quiet --prompt '帮我生成本周工作总结'"
```

## 注意事项

1. **首次使用**：添加定时任务前，需要先向机器人发送至少一条消息，以便保存聊天上下文
2. **命令执行**：任务在机器人运行的服务器上执行，可以使用任何可用的命令
3. **结果发送**：任务执行结果会自动发送到创建任务时的聊天窗口
4. **持久化**：任务会自动保存，重启机器人后任务仍然有效
5. **时区**：使用服务器本地时区（通常是 UTC+8）

## 故障排查

### 任务没有按时执行

1. 检查任务是否启用：`/task list`
2. 检查机器人是否在线
3. 查看服务器日志

### 任务执行失败

1. 使用 `/task run <任务ID>` 手动测试
2. 检查命令是否在服务器上可用
3. 查看错误消息（会发送到聊天中）

### 添加任务失败

1. 确保 crontab 表达式格式正确
2. 确保命令不为空
3. 确保已向机器人发送过消息（有聊天上下文）
